# 게시판 개선 프로젝트 계획

## 목표 (Goal)
현재 기본 기능만 있는 로컬스토리지 기반 게시판을 실제 서비스 수준으로 개선하여, 사용자 경험을 높이고 보안 취약점을 제거한다.

---

## 질문 (Questions)

1. **서비스 범위**  
   - 이 게시판을 외부에 공개할 수 있는 서비스로 만들어야 하나요, 아니면 학습/테스트용인가요?

2. **데이터 규모**  
   - 예상 게시글 수가 몇 개까지 예상되나요? (수십 개 vs 수백 개)

3. **우선순위**  
   - UI/UX 개선(디자인)과 기능 개선(검색, 정렬, 페이징) 중 어느 쪽이 더 중요한가요?

---

## 범위 (Scope - Steps)

### Step 1: 보안 취약점 제거 (필수)
- 비밀번호 평문 저장 문제 해결 (해싱 처리)
- localStorage 데이터 암호화 검토

### Step 2: 사용성 개선 (우선순위 높음)
- 게시글 검색 기능 (제목, 내용, 작성자 검색)
- 정렬 옵션 (최신순, 조회수순, 추천순)
- 페이지네이션 (20개 단위 페이징)
- 게시글 미리보기 길이 개선

### Step 3: 메타데이터 추가 (선택)
- 조회수 (views) 카운팅
- 좋아요/추천 기능
- 카테고리 분류
- 댓글 삭제 기능 (작성자만)
- 게시글 수정 시간 기록

---

## 인수 조건 (Acceptance Criteria)

1. **보안**
   - [ ] 비밀번호가 해싱되어 저장됨
   - [ ] 로그인 시 해시 비교로 인증됨

2. **검색 및 필터링**
   - [ ] 검색어로 제목/내용/작성자 검색 가능
   - [ ] 정렬 옵션(최신순, 오래된순) 선택 가능

3. **성능**
   - [ ] 게시글 20~30개 이상에서도 페이지 로드 지연 없음
   - [ ] 페이지네이션 구현 완료

4. **사용성**
   - [ ] 게시글 조회수 표시
   - [ ] 댓글 작성자만 댓글 삭제 가능
